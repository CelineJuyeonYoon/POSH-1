### Commit 규칙

- commit -m type subject(한글)
- type 지정 
    - ex) Add : 로그인페이지 UI추가
 1. Fix: 버그수정
 2. Add: 코드추가
 3. Remove: 코드삭제
 4. Chore: 패키지 수정/삭제
 5. Update: 자잘한 수정 

### PR 규칙

- 본인이 작업한 Feature(기능) Branch에 작업한 최소단위 커밋으로 PR 요청합니다.
- 해당 PR의 유효성 검토 후 Develop Branch에 담당자를 통한 Merge가 이뤄집니다.
